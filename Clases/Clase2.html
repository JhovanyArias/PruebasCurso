Etiquetas básicas.

  • <table> - Define toda la tabla y sirve como el contenedor del contenido.
  • <tr> - La fila de la tabla (table row). No puede estar vacío, debe contener al menos una celda.
  • <td> - La celda de tabla (table data). Puede ser usada solo dentro de una fila. Cada fila de la tabla debe tener el mismo número de las celdas.

Celdas de encabezado

Para crear una fila con encabezados de columnas se usa la etiqueta estándar <tr> para la fila y marcamos las celdas con una etiqueta especial <th> (table header) - que es la etiqueta semántica de la celda de encabezado. De forma predeterminada, el texto dentro de <th> se pone en negrita y se centra.

Secciones de la tabla

Hay varias etiquetas estructurales para mejorar la semántica del marcado de la tabla.

  • <thead> - Es el membrete que envuelve un grupo de filas de tabla en el área de encabezado.
  • <tbody> - Es el cuerpo que envuelve el grupo principal de filas de la tabla.
  • <tfoot> - Es el sótano que denota un grupo de filas de tabla en el área del pie de página.

El título de la tabla

Si la tabla tiene un encabezado se puede marcarlo usando etiquetas de encabezado pero sería más semántico usar una etiqueta de tabla especial <caption>, que debe ubicarse como el primer elemento dentro de la tabla

Agrupación de celdas

Para expandir una celda horizontalmente, se usa el atributo -colspan, verticalmente - rowspan.

Diseño del texto

  • La etiqueta <dl> (description list): Es un contenedor para una lista de pares de términos y sus descripciones, es decir, es un diccionario. Para el marcado de términos hay una etiqueta dt, y para descripciones se usa la etiqueta dd

  • La etiqueta <span>: Es un contenedor universal para el contenido textual pequeño, como palabras, partes de palabras o frases completas dentro de un texto. De manera similar a la etiqueta div, no tiene ningún significado semántico y se utiliza para el estilo adicional

  • La etiqueta <pre>: De forma predeterminada, el navegador ignora los saltos de línea y más de un espacio en una fila. Esta etiqueta le permite agregar texto preformateado con saltos de línea forzados. Por ejemplo, marcar el texto de una canción o un poema.

  • Las etiquetas <sup> y <sub>: Sirven para mostrar los símbolos en el índice superior (sup, es la abreviatura de superscript) e inferior (sub, es la abreviatura de subscript).

  • La etiqueta <time>: La fecha y la hora pueden ser escritas simplemente con texto, pero es mejor usar una etiqueta especial <time>:
    La sesión tuvo lugar en time datetime="2018-07-14T11:30">11:30 por la mañana
    El navegador solo mostrará el texto dentro de la etiqueta. La persona verá el texto comprensible y la tecnología asistencial leerá el atributo datetime y obtendrá el valor exacto en el formato deseado.

  • La etiqueta <address>: Sirve para guardar la información de contacto sobre el autor de un artículo, un bloque de información o una página web en general

  • Las etiquetas <strong> y <b>: Se utilizan para resaltar una palabra importante o una frase completa dentro del texto. Visualmente, ambas etiquetas hacen que el texto sea audaz y, a primera vista, no hay diferencia entre ellas, pero no es así.
    La etiqueta <strong> enfatiza semánticamente el texto como importante e indica que es una palabra o frase significativa. Por ejemplo, resaltar una advertencia o cualquier texto al que el usuario deba prestar atención en primer lugar.
    La etiqueta <b> (es abreviatura de bold) hace el texto visualmente más audaz, atrae la atención pero sin darle mucha importancia.

  • Las etiquetas <em> e <i>: Se utilizan para enfatizar partes del texto. Visualmente, ambas etiquetas hacen que el texto sea cursivo y, a primera vista, no hay diferencia entre ellas, pero no es así.
    La etiqueta <em> (es abreviatura de emphasis) resalta semánticamente el texto en el que se pone especial hincapié que cambia el significado de la frase. Por ejemplo, se puede asignar el sarcasmo.
    La etiqueta <i> (es abreviatura de italic) hace que el texto solo sea visualmente cursivo, atrae la atención pero sin darle mucha importancia. Por ejemplo, resaltar términos en una oración.

Los caracteres comodines

  • No todos los caracteres pueden ser escritos en el teclado. Por ejemplo, el símbolo de marca registrada ®, del nombre comercial ™, derechos de autor © se utilizan los caracteres comodines.

Audio y vídeo

  • Vídeo: Para marcar el contenido de vídeo se usa la etiqueta de relación <video>
    <video
     src="http://techslides.com/demos/sample-videos/small.webm"
     poster="https://images.pexels.com/photos/595804/pexels-photo-595804.jpeg?w=640"
     width="640"
     controls
     autoplay
     loop
     preload="auto"
    ></video>
      Vamos a analizar los atributos.
    • src - Especifica la dirección de vídeo
    • poster - La dirección de la imagen que se utiliza como póster si el vídeo no se ha descargado o aún no se está reproduciendo
    • width y height - Establece la altura y el ancho como la imagen
      • controls - Atributo comprobado, si está presente, se agregan los campos de control de vídeo
    • autoplay - Atributo comprobado, si está presente, la reproducción de vídeo comienza tan pronto como esté disponible
      • loop - Atributo comprobado, si está presente, el vídeo se repite en un ciclo
      El atributo preload señala el modo de precarga del video. Su valor afecta a la información que se cargará. El valor predeterminado depende del navegador en el que se abrió la página web.
    • none - No subir videos de antemano
    • metadata - Cargar información oficial (duración, etc.)
    • auto - Cargar todos los vídeos de antemano

  • Formatos de vídeo: No es suficiente descargar solo un formato de archivo de video porque es posible que algunos navegadores no puedan soportarlo y que el usuario no pueda ver el vídeo. Hay varios formatos de video populares que debe proporcionar al navegador para elegir: webm, mp4 y ogg.
    Para hacer esto, eliminamos el atributo src de la etiqueta <video> y agregamos una nueva etiqueta <source> para cada formato. En el atributo src especificamos la dirección del archivo de vídeo y en el atributo type el formato de vídeo. El navegador seleccionará el primero de la lista que admite y lo descargará, ignorando el resto.
    <video
     poster="https://images.pexels.com/photos/595804/pexels-photo-595804.jpeg?w=640"
     width="640"
     controls
    >
     <source
     src="http://techslides.com/demos/sample-videos/small.webm"
     type="video/webm"
     />
     <source
     src="http://techslides.com/demos/sample-videos/small.mp4"
     type="video/mp4"
     />
     <source
     src="http://techslides.com/demos/sample-videos/small.ogg"
     type="video/ogg"
     />
    </video>

  • Audio: Para marcar el contenido de audio, se utiliza la etiqueta de relación <audio>. Su uso es muy similar a <video>, lo único es que se puede especificar menos atributos.
    <audio src="song.mp3" controls autoplay loop preload="auto"></audio>
      Vamos a analizar los atributos.
    • src - Especifica la dirección del de audio
    • controls - Atributo comprobado, si está presente, se agregan controles de
    • audio
    • autoplay - Atributo comprobado, si está presente, la reproducción de audio comienza tan pronto como esté disponible
    • loop - Atributo comprobado, si se aplica, el audio se repite en un ciclo
      • El atributo preload especifica el modo de precarga del archivo de audio. Su significado afecta a qué información se cargará. El valor predeterminado depende del navegador en el que se abrió la página web.
    • none - No descargar audio de antemano
    • metadata - Cargar la información oficial (duración, etc.)
    • auto - Cargar de antemano todo el archivo de audio
  • Formatos de audio: Con el soporte de formatos en el navegador, la situación de audio es mejor. Todos los navegadores modernos están garantizados para admitir uno de los dos formatos: mp3 o ogg. Al igual que con el video, eliminamos el atributo src y usamos la etiqueta <source> con los atributos src y type.
    <audio controls>
     <source src="song.mp3" type="audio/mpeg" />
     <source src="song.ogg" type="audio/ogg" />
    </audio>
